2. Data attached to objects

Objects displayed by the canvas can have some data associated for identification , or to add some feature. For example, all objects can have a name. This name is set by:

void evas_object_set_name(Evas canvas, Evas_Object object, char *name);

By default, Evas objects have no name. The 'name' parameter is internally copied, so that if an object's name is changed the older name is freed. To remove an object's name, simply call this function with 'name' being NULL.

Inversely, it is possible to read an object's name:

char *evas_object_get_name(Evas canvas, Evas_Object object);

If the object is nameless, the function returns NULL.

But the really interesting feature is that an object on a canvas can be found by its name:

Evas_Object evas_object_get_named(Evas canvas, char *name);

When this function is called, Evas will look on canvas 'canvas' the first object whose name is 'name'. It is thus better to use different names for each object. If no object is found, the function returns NULL.

It is also possible to add much more generic data to Evas objects. This feature is key-based: to each data is associated an alphanumeric key which identifies the data for future retrieval. To associate a key/data couple to an Evas object, use this function:

void evas_put_data(Evas canvas, Evas_Object object,
                   char *key, void *data);

The object's data, associated to a given key, can afterwards be retrieved by this function:

void *evas_get_data(Evas canvas, Evas_Object object, char *key);

If no data is found that corresponds to the 'key' key for the 'object' object, the return value is NULL.

A key and its data can be unlinked for a given object by this function:

void *evas_remove_data(Evas canvas, Evas_Object object, char *key);

All this can be seen has a hash table attached to each object.
