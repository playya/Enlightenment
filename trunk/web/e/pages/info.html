Information

Enlightenment is a window manager for X. Its design goal is to be as configurable as possible - in look AND in feel.

Enlightenment's current design aim is to become a desktop shell. That means it will manage your application windows, being able to launch applications, and also manage your files.

We're currently hard at work building the next revision (0.17) of Enlightenment.

Frequently Asked Questions

This collection is currently being worked on. Bear with us if what you're looking for isn't here yet. In that case, please ask on the mailing lists instead or check the archives.

General

What exactly do you mean by "E 17 will become a desktop shell?"
It means that E 17 combines features of a window manager and a file manager. It will provide nicely integrated GUI elements for managing your desktop elements, both files and windows. It does *not* mean that E 17 is another application framework like Gnome and KDE.

Will E 16's themes work with E 17?
No. The theme formats are totally different. There is a chance though that converters will exist at some point.

What's E's relation to Gnome and KDE? Is Gnome supported? Is KDE supported?
E 16 supports both Gnome and KDE. In E 17, you will be able to use both Gnome and KDE *applications*. However, given that E 17 will be a desktop shell, this doesn't mean that all parts of the frameworks (such as desktop-oriented filemanagers) will work.

What platforms does E run on?
The primary development platform is Linux. *BSD and Solaris are also supported. HP, Irix, AIX should more or less work, but are probably adventurous.

I'd like to contribute! How can I commit code?
Send patches! If they're good you'll get a CVS account.

E 16

My epplets keep multiplying whenever I launch E. Help!
That's a bug in the Epplet library. For the adventurous, you need to remove all bogus epplet entries from ~/.enlightenment/...e_session-XXXXXX.snapshots.0 while E isn't running. A more radical approach is to simply delete this file -- this may cause other settings to be lost, such as those for the icon box and the pagers. Sorry for the inconvenience.

Efsd

I heard Efsd needs FAM. What's that? What's Imon?
FAM is the File Alteration Monitor, a server which reports file modification events to clients. Imon is a Linux kernel patch to improve performance, using it is highly recommended. Read more about it at SGI.

Can I run Efsd without FAM? Do I need Imon?
At the moment, FAM is necessary to run Efsd. You can get both FAM and Imon at SGI. If the server is down, we have a local copy of FAM's sources up on our SourceForge files page. This version also fixes an annoying compilation bug.

I can't compile FAM. It dies with a "`NULL' undeclared (first use this function)" message.
Add the line "#include <stdlib.h>" to BTree.h. Alternatively, you can download our copy which has this problem fixed on our SourceForge files page.

Why don't you add support for browsing FTP, Tarballs etc to Efsd?
Because that's not Efsd's job. Efsd's job is to provide a facility for launching fs commands and asynchronously receiving replies *on top* of the fs, not to extend the fs.
